# စီမံကိန်းအမည်: မြန်မာစကားလုံးဖြတ်တောက်ခြင်း တည်းဖြတ်ဆော့ဖ်ဝဲ (Myan-Seg-Editor) ဒီဇိုင်းသတ်မှတ်ချက်

## ၁။ စီမံကိန်းခြုံငုံသုံးသပ်ချက် (Project Overview)


Myan-Seg-Editor သည် ထိရောက်ပြီး ပဋိပက္ခကင်းစင်သော မြန်မာစကားလုံး/စကားစု ဖြတ်တောက်ခြင်း (Word/Phrase Segmentation) မှတ်သားမှုပြုလုပ်သည့် ကိရိယာတစ်ခု တည်ဆောက်ရန် ရည်ရွယ်သည်။ ဤကိရိယာ၏ အဓိကရည်မှန်းချက်မှာ အရည်အသွေးမြင့် မှတ်သားထားသော ဒေတာအစုအဝေးများ (Annotated Datasets) ထုတ်လုပ်ရန်အတွက် စာလုံးအဆင့် ဖြတ်တောက်ခြင်းအပေါ် အခြေခံ၍ **ကိုက်ညီမှုရှိခြင်း (Consistency)** နှင့် **မြန်ဆန်ခြင်း (Speed)** ကို ဦးစားပေးရန်ဖြစ်သည်။

## ၂။ အဓိကအတွေးအခေါ် (Core Philosophy)

* **ဖြတ်တောက်သည့်ယူနစ် (Segmentation Unit):** စာလုံး (Character) အဆင့်ထက် ဝဏ္ဏ (Syllable) အဆင့်ကို အခြေခံသည်။
* **လုပ်ငန်းစဉ် (Workflow):** အစမှအဆုံး ကိုယ်တိုင်ဖြတ်တောက်ခြင်းထက် AI အသုံးပြု၍ ကြိုတင်ဖြတ်တောက်ပြီးမှ ပြန်လည်ပြင်ဆင်ခြင်း (Post-editing mode) ကို ဦးစားပေးသည်။
* **အရည်အသွေးထိန်းချုပ်မှု (Quality Control):** လိုအပ်သည်ထက် ပိုမိုဖြတ်တောက်ခြင်း သို့မဟုတ် လျော့နည်းဖြတ်တောက်ခြင်းများကို ရှောင်ရှားရန် အချိန်နှင့်တပြေးညီ ပဋိပက္ခစစ်ဆေးခြင်း (Real-time conflict detection) ကို အသုံးပြုသည်။

## ၃။ အဓိကလုပ်ဆောင်ချက်များ (Key Features)

### ၃.၁။ တည်းဖြတ်မှုယန္တရား (Editor Mechanism)

* **စနစ်နှစ်မျိုးအသုံးပြုခြင်း (Dual Mode System):**
  1. **ဖြတ်တောက်ခြင်းစနစ် (Segmentation Mode):** ပုံသေစနစ်ဖြစ်သည်။ ဝဏ္ဏများကို ရွှေ့ရန် arrow keys များ၊ ဖြတ်ရန် `Space` key နှင့် ပေါင်းရန် `Backspace` key တို့ကို အသုံးပြုသည်။
  2. **ပြင်ဆင်ခြင်းစနစ် (Edit Mode):** Double-click နှိပ်ခြင်း သို့မဟုတ် shortcut (ဥပမာ `F2`) ဖြင့် အသုံးပြုနိုင်သည်။ စာလုံးပေါင်းမှားယွင်းမှုများကို ပြင်ဆင်ရန် (စာလုံးအဆင့် တည်းဖြတ်ခြင်း) အသုံးပြုသည်။ ဤစနစ်တွင် cursor သည် စာလုံးတစ်လုံးချင်းစီ ရွှေ့နိုင်ပြီး သာမန်စာစီစာရိုက်စနစ်ကဲ့သို့ ပြင်ဆင်နိုင်သည်။ ပြင်ဆင်ပြီးနောက် `Enter` နှိပ်ခြင်း သို့မဟုတ် အခြားနေရာတွင် click နှိပ်ခြင်းဖြင့် ဖြတ်တောက်မှုဆိုင်ရာ logic (Regex) ကို ချက်ချင်းပြန်လည်လုပ်ဆောင်စေပြီး ဝဏ္ဏများကို ပြန်လည်တည်ဆောက်ကာ ဖြတ်တောက်ခြင်းစနစ်သို့ အလိုအလျောက် ပြန်ပြောင်းသွားမည်ဖြစ်သည်။
* **စမတ်ကျသော Logic (Smart Logic):** ဖြတ်တောက်ခြင်းနှင့် ပြင်ဆင်ခြင်းစနစ်များအကြား အလိုအလျောက်ကူးပြောင်းရန် Regex/Sylbreak logic ကို အသုံးပြုသည်။

### ၃.၂။ ကိုက်ညီမှုနှင့် အရည်အသွေးအာမခံချက် (Consistency & Quality Assurance)

* **တစ်ကမ္ဘာလုံးဆိုင်ရာ ဝေါဟာရစာရင်း (Global Glossary):** အသုံးပြုသူ၏ ဖြတ်တောက်မှု ဆုံးဖြတ်ချက်များကို မှတ်တမ်းတင်ထားသည်။
  * **အဆင့်လိုက်မှတ်ဉာဏ်စနစ် (Tiered Memory System):**
    * **ကိုယ်တိုင်ထည့်သွင်းသည့်မှတ်ဉာဏ် (Manual Memory):** အသုံးပြုသူများသည် ရှားပါးစကားလုံးများနှင့် နည်းပညာဆိုင်ရာ အသုံးအနှုန်းများကို `Right Click > Add to Dictionary` သို့မဟုတ် shortcuts များမှတစ်ဆင့် တိုက်ရိုက်ထည့်သွင်းနိုင်သည်။
    * **အလိုအလျောက်မှတ်ဉာဏ် (Auto Memory):** အသုံးပြုသူ ပြင်ဆင်ပြီးနောက် စကားလုံးများသည် ယာယီ cache ထဲသို့ ရောက်ရှိသွားသည်။ အကယ်၍ ထိုစကားလုံးသည် ဒေတာအစုအဝေးတွင် ၃ ကြိမ်ထက်မက တူညီသောဖြတ်တောက်မှုမျိုး တွေ့ရှိပါက အမြဲတမ်းအဘိဓာန် (Permanent Dictionary) ထဲသို့ အလိုအလျောက် ရွှေ့ပြောင်းပေးမည်ဖြစ်သည်။
* **ပဋိပက္ခစစ်ဆေးခြင်း (Conflict Detection):**
  * ဒေတာအစုအဝေးအတွင်း `A B` (ဖြတ်ထားခြင်း) နှင့် `AB` (ပေါင်းထားခြင်း) အကြား မကိုက်ညီမှုများကို ရှာဖွေသည်။
  * **အခြေအနေအလိုက် အလိုအလျောက်ပျံ့နှံ့စေခြင်း (Context-aware Auto-propagation):** မှားယွင်းသော အလိုအလျောက်အစားထိုးမှုများကို ရှောင်ရှားရန် N-gram အခြေအနေပေါ်မူတည်၍ ခွဲခြားပေးသည်။
  * **သံသယဖြစ်ဖွယ်ခွဲခြားခြင်း (Ambiguity Classification):** စကားလုံးများကို "ဘေးကင်းသောစကားလုံးများ" (အလိုအလျောက် ပြောင်းလဲနိုင်သည်) နှင့် "သံသယဖြစ်ဖွယ်စကားလုံးများ" (ကိုယ်တိုင်ပြန်လည်စစ်ဆေးရန်လိုအပ်သည်) ဟု ခွဲခြားသတ်မှတ်သည်။
* **အမြင်အာရုံဆိုင်ရာ သတိပေးချက်များ (Visual Warnings):**
  * ပုံမှန်ထက် ပိုရှည်နေသော ဝဏ္ဏများကို (ဖြတ်တောက်ရန် ကျန်ရှိနေနိုင်သည်ဟု ယူဆကာ) အရောင်ဖြင့် ပြသပေးသည်။
  * **ပဋိပက္ခစောင့်ကြည့်ကိရိယာ (Conflict Monitor):** Sidebar တွင် သတိပေးချက်များ ပြသပေးသည်၊ ဥပမာ - "သတိပေးချက်! 'ကျောင်းသား' ဆိုသော စကားလုံးကို စာကြောင်း ၅၀ တွင် ခွဲရေးထားသည်၊ ဤနေရာတွင် ပေါင်းရမည်လား?"
* **လုပ်ဆောင်ချက်ပြန်ပြင်ခြင်း (Undo History):** အလိုအလျောက် ပြောင်းလဲမှုများကို ပြန်ပြင်ရန် `Ctrl+Z` ကို အသုံးပြုနိုင်သည်။

### ၃.၃။ ဒေတာသန့်စင်ခြင်း (Data Cleaning/Preprocessing)

* **မမြင်ရသော စာလုံးများဖယ်ရှားခြင်း (Invisible Characters Removal):** Zero-Width Space (ZWSP, ``), Zero-Width Non-Joiner (ZWNJ, `‌`) နှင့် carriage returns (``) ကဲ့သို့သော မလိုအပ်သည့် Unicode control characters များကို ဖယ်ရှားသည်။
* **Space များ ပုံမှန်ဖြစ်စေခြင်း (Space Normalization):** Space အများအပြားကို Space တစ်ခုတည်းအဖြစ် အစားထိုးပြီး ရှေ့နှင့်နောက်တွင်ရှိသော Space များကို ဖြတ်ထုတ်သည်။
* **ကြည့်ရှုမှုရွေးချယ်စရာများ (View Options):** စနစ်နှစ်မျိုးပေးထားသည် - စာကြောင်းအလိုက်ကြည့်ခြင်း (Sentence-level View - NLP အတွက် သင့်တော်သည်) နှင့် စာပိုဒ်အလိုက်ကြည့်ခြင်း (Paragraph View - မူရင်းစာပိုဒ်ပုံစံအတိုင်း ထားရှိသည်)။

### ၃.၄။ အနုစိတ်ထိန်းချုပ်မှုနှင့် စမတ်ကျသော စုစည်းမှု Logic (Granularity Control and Smart Grouping Logic)

မတူညီသော အလုပ်များအတွက် လိုအပ်သည့် ဖြတ်တောက်မှုအနုစိတ်နှုန်းကို လိုက်လျောညီထွေဖြစ်စေရန်၊ ဤတည်းဖြတ်ဆော့ဖ်ဝဲသည် သဒ္ဒါဆိုင်ရာ စုစည်းမှုစည်းမျဉ်းများကို စိတ်ကြိုက်ပြင်ဆင်နိုင်သည်-

* **စည်းမျဉ်းအခြေခံ ပြင်ဆင်သတ်မှတ်ခြင်း (Rule-based Configuration):** သတ်မှတ်ထားသော သဒ္ဒါစည်းမျဉ်းများကို ပေါင်းမည် သို့မဟုတ် ခွဲမည်ကို project settings တွင် ခလုတ်များဖြင့် ထိန်းချုပ်နိုင်သည်။
  * **ဝိဘတ်များ (Postpositional Markers - ADP):** ဥပမာ `က၊ ကို၊ မှာ`။ ရွေးချယ်စရာ - "အမြဲခွဲရန်" သို့မဟုတ် "နာမ်နှင့်ပေါင်းရန်"။
  * **အများကိန်းပြပစ္စည်းများ (Plural Markers):** ဥပမာ `များ၊ တို့၊ တွေ`။ ရွေးချယ်စရာ - "အမြဲခွဲရန်" သို့မဟုတ် "နာမ်နှင့်ပေါင်းရန်"။
  * **ကာလပြပစ္စည်းများ (Tense/Aspect Markers):** ဥပမာ `သည်၊ မည်၊ ပြီ၊ ခဲ့`။ ရွေးချယ်စရာ - "အမြဲခွဲရန်" သို့မဟုတ် "ကြိယာနှင့်ပေါင်းရန်"။
  * **ကြိယာဝိသေသနနောက်ဆက်တွဲများ (Adverbial Suffix Grouping):** ဥပမာ `စွာ`။ ရွေးချယ်စရာ - "ပေါင်းရန်" သို့မဟုတ် "ခွဲရန်"။
  * **ငြင်းပယ်ခြင်း (Negation Grouping):** ဥပမာ `မ` + ကြိယာ။ ရွေးချယ်စရာ - "ပေါင်းရန်" သို့မဟုတ် "ခွဲရန်"။
  * **နာမဝိသေသနပြုပစ္စည်း (Relative Participle):** ဥပမာ `သော/သည့်/မည့်`။ ရွေးချယ်စရာ - "ပေါင်းရန်" သို့မဟုတ် "ခွဲရန်"။
* **ကြိုတင်သတ်မှတ်ထားသော ပရိုဖိုင်များ (Preset Profiles):** အမြန်ရွေးချယ်နိုင်ရန် "စကားလုံးအဆင့်" (Word Level)၊ "စကားစုအဆင့်" (Phrase Level) နှင့် "ဝဏ္ဏအဆင့်" (Syllable Level) ဟူ၍ ကြိုတင်ပြင်ဆင်ထားသည်။
* **N-gram အကြံပြုစနစ် (N-gram Suggestion System):** ဝီကီပီးဒီးယားကဲ့သို့သော ဒေတာအစုအဝေးကြီးများမှ စာရင်းအင်းအချက်အလက်များပေါ်မူတည်၍ တွဲလျက်ရှိသော ဝဏ္ဏများ (ဥပမာ `("ပီပြင်", "စွာ")`) အတွက် ပေါင်းစည်းရန် အကြံပြုချက်များ ပေးသည်။ ပေါင်းစည်းနိုင်ခြေ သတ်မှတ်ချက်ထက် ကျော်လွန်ပါက စနစ်က အလိုအလျောက် ပေါင်းပေးခြင်း သို့မဟုတ် အကြံပြုခြင်းများ ပြုလုပ်နိုင်သည်။
  * **UI အကောင်အထည်ဖော်မှု:** "N-gram ဖြင့် အလိုအလျောက်စုစည်းရန်" ခလုတ်တစ်ခု ပါဝင်မည်။ နှိပ်လိုက်ပါက စနစ်က ဖြစ်နိုင်ခြေမြင့်မားသော ပေါင်းစပ်စကားလုံးများကို အလိုအလျောက် ပေါင်းစည်းပေးမည်ဖြစ်သည်။

## ၄။ နည်းပညာပိုင်းနှင့် Algorithms (Tech Stack & Algorithms)

* **ဖြတ်တောက်ခြင်း Algorithm (Segmentation Algorithm):** စွမ်းဆောင်ရည်မြင့်မားပြီး ပွင့်လင်းမြင်သာမှုရှိစေရန် ပြုပြင်ထားသော `Sylbreak` (Regex-based) algorithm ကို အသုံးပြုသည်။ Backend ပိုင်းအတွက် `myanmar-tools` သို့မဟုတ် `pyidaungsu` ကဲ့သို့သော machine learning/graph-based libraries များကို ထည့်သွင်းစဉ်းစားနိုင်သည်။
* **Backend:** Python (ဖြတ်တောက်ခြင်းဆိုင်ရာ logic များအတွက် အထူးသင့်လျော်သည်)။
* **Frontend:**
  * **နမူနာပုံစံအမြန်ထုတ်ခြင်း (Rapid Prototyping):** Streamlit (Python ဖြင့် UI အမြန်တည်ဆောက်ရန်)။
  * **ပိုမိုကောင်းမွန်သော User Interface:** React.js သို့မဟုတ် Vue.js။
  * **မှတ်သားမှုပြုလုပ်သည့် Framework (Annotation Framework):** Label Studio သို့မဟုတ် Doccano ကဲ့သို့သော စိတ်ကြိုက်ပြင်ဆင်နိုင်သည့် လက်ရှိကိရိယာများ။

## ၅။ UI Layout ဒီဇိုင်း

တည်းဖြတ်ဆော့ဖ်ဝဲ၏ interface ကို အဓိကအပိုင်း ၃ ပိုင်း ခွဲခြားထားသည်-

* **[A] အပေါ်ဘက် Toolbar (Top Toolbar):**
  * **ဖိုင်လုပ်ဆောင်ချက်များ:** `Import Raw Text`, `Export Dataset`, `Save Progress`။
  * **သန့်စင်ခြင်းကိရိယာများ:** `Clean ZWSP`, `Normalize Spaces`, `Remove Double Spaces`။
  * **ကြည့်ရှုမှုရွေးချယ်စရာများ:** `Toggle Line/Paragraph View`, `Show/Hide Confidence Colors`။
* **[B] အဓိက လုပ်ငန်းခွင် ("Block" View):**
  * စာသားများကို **အပြန်အလှန်သက်ရောက်မှုရှိသော အကွက်များ (Interactive Blocks)** အဖြစ် ပြသသည်၊ ဥပမာ - `[ ကျောင်း ] [ သား ] [ များ ]`။
  * **အမြင်အာရုံပုံစံ:** Cursor ရောက်နေသော အကွက်ကို **အပြာရောင်ဘောင်** ဖြင့် ပြသပြီး ပဋိပက္ခရှိနေသော အကွက်များကို **အဝါရောင်နောက်ခံ** ဖြင့် ပြသသည်။
  * **အပြန်အလှန်သက်ရောက်မှု:** အကွက်နှစ်ခုအကြား ပေါင်းလိုပါက `Backspace` နှိပ်ပါ (`[ ကျောင်းသား ]`)၊ အကွက်တစ်ခုအတွင်း ခွဲလိုပါက `Space` နှိပ်ပါ (`[ ကျောင်း ] [ သား ]`)။
* **[C] ညာဘက်ရှိ ဉာဏ်ရည်တု Sidebar (Right Intelligence Sidebar):**
  * **ပဋိပက္ခစောင့်ကြည့်ကိရိယာ (Conflict Monitor):** ပဋိပက္ခသတိပေးချက်များနှင့် အကြံပြုချက်များကို ပြသသည်။
  * **အဘိဓာန်ရှာဖွေခြင်း (Dictionary Lookup):** လက်ရှိရွေးချယ်ထားသော စကားလုံးသည် အဘိဓာန်ထဲတွင် ရှိမရှိ ပြသသည်။
  * **စာရင်းအင်းများ (Statistics):** မှတ်သားမှုတိုးတက်မှုကို ပြသသည် (ဥပမာ - "ပြီးစီးမှု - ၁၅ / ၁၀၀ ကြောင်း")။

## ၆။ ဒေတာတည်ဆောက်ပုံ ဒီဇိုင်း (Data Structure Design)

ကိုက်ညီမှုရှိစေရန်နှင့် metadata များ သိမ်းဆည်းရန် ဒေတာများကို **JSON format** ဖြင့် သိမ်းဆည်းရန် အကြံပြုသည်။

**JSON Format ဥပမာ:**

```json
{
  "project_meta": {
    "name": "Medical_Dataset_01",
    "created_at": "2025-07-12",
    "total_lines": 1000
  },
  "content": [
    {
      "id": 1,
      "original_text": "ဆရာဝန်လာပါပြီ",
      "segments": ["ဆရာဝန်", "လာ", "ပါ", "ပြီ"],
      "status": "reviewed", 
      "flags": [] 
    },
    {
      "id": 2,
      "original_text": "ခွဲစိတ်ကုသမှုအောင်မြင်သည်",
      "segments": ["ခွဲစိတ်", "ကုသ", "မှု", "အောင်မြင်", "သည်"],
      "status": "conflict",
      "flags": ["under_segmentation_warning"]
    }
  ],
  "glossary_updates": ["ခွဲစိတ်", "ကုသမှု"]
}
```

* **အားသာချက်များ:** နောင်တွင် algorithm များအား ပြန်လည်စစ်ဆေးရန် `original_text` ကို ထိန်းသိမ်းထားနိုင်ခြင်း၊ `status` နှင့် `flags` အကွက်များက မှတ်သားမှုလုပ်ငန်းစဉ်များကို စီမံခန့်ခွဲရာတွင် ကူညီပေးခြင်း။

## ၇။ ပဋိပက္ခဖြေရှင်းမှု လုပ်ငန်းစဉ် (Conflict Resolution Flow)

အသုံးပြုသူက ဖြတ်တောက်မှု ပြင်ဆင်လိုက်သည့်အခါ စနစ်သည် အောက်ပါ အဆင့် ၄ ဆင့်ကို လုပ်ဆောင်မည်ဖြစ်သည်။

### ၇.၁။ စတင်ခြင်းနှင့် ဒေသတွင်းလုပ်ဆောင်ချက် (Trigger & Local Action)

အသုံးပြုသူသည် တည်းဖြတ်မှုအပိုင်းတွင် `Space` (ခွဲရန်) သို့မဟုတ် `Backspace` (ပေါင်းရန်) ကို နှိပ်သည်။ Interface တွင် ပြောင်းလဲမှုများကို ချက်ချင်းပြသသည်။

### ၇.၂။ နောက်ကွယ်မှ စစ်ဆေးခြင်း (Background Scan)

စနစ်သည် အသုံးပြုသူ ပြင်ဆင်လိုက်သော စကားလုံး (`New_Word`) အပေါ် အခြေခံ၍ ဒေတာအစုအဝေးတစ်ခုလုံးနှင့် ဝေါဟာရစာရင်းကို စစ်ဆေးသည်။

* **ပြောင်းပြန်ရှာဖွေခြင်း (Reverse Search):** အသုံးပြုသူက ပေါင်းလိုက်လျှင် ဒေတာအစုအဝေးထဲတွင် ခွဲထားသည့် စကားလုံးများကို ရှာသည်၊ အသုံးပြုသူက ခွဲလိုက်လျှင် ပေါင်းထားသည့် စကားလုံးများကို ရှာသည်။
* **ဝေါဟာရစာရင်းစစ်ဆေးခြင်း (Glossary Check):** အဘိဓာန်ထဲတွင် သတ်မှတ်ထားသော ပဋိပက္ခများကို စစ်ဆေးသည်။

### ၇.၃။ ပဋိပက္ခဆုံးဖြတ်ချက် Logic (Conflict Decision Logic)

�စ်ဆေးမှုရလဒ်ပေါ်မူတည်၍ စနစ်သည် အောက်ပါ အခြေအနေ ၃ ခုအနက်မှ တစ်ခုကို ရွေးချယ်သည်-

* **အခြေအနေ (က) ပဋိပက္ခမရှိခြင်း (No Conflict):** ဒေတာအစုအဝေးနှင့် ဝေါဟာရစာရင်းထဲတွင် အခြားပဋိပက္ခဖြစ်နိုင်သော ပုံစံများ မရှိပါ။ စကားလုံးကို **ယာယီဝေါဟာရစာရင်း** ထဲသို့ ထည့်သွင်းပြီး လုပ်ငန်းစဉ်ပြီးဆုံးသည်။
* **အခြေအနေ (ခ) ယုံကြည်မှုမြင့်မားသော ပဋိပက္ခ (High Confidence Conflict):** ဝေါဟာရစာရင်းတွင် ပုံစံတစ်ခုကို ရှင်းလင်းစွာ သတ်မှတ်ထားသော်လည်း ဒေတာအ�ုအဝေးတွင် မကိုက်ညီမှုများ ရှိနေသည်။ စနစ်က အသုံးပြုသူအား **အလိုအလျောက်ပျံ့နှံ့စေရန် (Auto-Propagation)** မေးမြန်းမည်ဖြစ်ပြီး မကိုက်ညီမှုအားလုံးကို တစ်ပြေးညီဖြစ်စေရန် လုပ်ဆောင်မည်ဖြစ်သည်။
* **အခြေအနေ (ဂ) သံသယဖြစ်ဖွယ် ပဋိပက္ခ (Ambiguity Conflict):** စကားလုံး၏ ပုံစံနှစ်မျိုးလုံး (ဥပမာ "စားပွဲ" နှင့် "စားပွဲ") သည် ဆီလျော်မှုရှိနေနိုင်သည်။ စနစ်က အလိုအလျောက် မပြင်ဆင်ဘဲ **"ပြန်လည်သုံးသပ်မှုစနစ်" (Review Mode)** ထဲသို့ ထည့်သွင်းမည်ဖြစ်သည်။

### ၇.၄။ ဖြေရှင်းမှု Interface (Resolution Interface)

ပဋိပက္ခတစ်ခု တွေ့ရှိပါက အောက်ပါ ရွေးချယ်စရာများပါဝင်သော pop-up window သို့မဟုတ် sidebar ပေါ်လာမည်-


| **ရွေးချယ်စရာ**                             | **အဓိပ္ပာယ်**                                                                                                           | **စနစ်၏ လုပ်ဆောင်ချက်**                                                                                                                                                                                                                                                                                                                |
| :----------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **၁။ အားလုံးပြင်ရန် (Apply Globally)** | "ယခင်က ဖြတ်တောက်မှု မှားယွင်းခဲ့သည်၊ ယခုပြင်ဆင်မှုကသာ မှန်ကန်သည်။"   | ဒေတာအစုအဝေးရှိ`A B` အားလုံးကို `AB` သို့ ချက်ချင်းပြောင်းသည်။ ဝေါဟာရစာရင်းတွင် `AB` ကို အမှန်အဖြစ် မှတ်သားသည်။                                                                                                                                                   |
| **၂။ ချွင်းချက် (Contextual)**             | "ဤနေရာတွင်သာ ပေါင်းမည်၊ အခြားနေရာများတွင် ခွဲထားသည်မှာ မှန်ကန်သည်။" | လက်ရှိနေရာတွင်သာ`AB` ကို ထားရှိသည်။ အခြားနေရာများကို မပြောင်းလဲပါ။ ဤစကားလုံးကို **"သံသယဖြစ်ဖွယ်စာရင်း" (Ambiguous List)** ထဲသို့ ထည့်လိုက်ပြီး နောက်ပိုင်းတွင် အလိုအလျောက် ပြောင်းလဲမည်မဟုတ်ပါ။ |

## ၈။ နည်းပညာဆိုင်ရာ လမ်းပြမြေပုံ (Technical Roadmap)

ဤစနစ်ကို အောက်ပါ အဆင့်ဆင့်အတိုင်း အကောင်အထည်ဖော်ရန် အကြံပြုသည်-
၁။ **တင်ပို့ခြင်းနှင့် သန့်စင်ခြင်း (Upload & Clean):** စာသားဖိုင်များတင်ခြင်း -> မမြင်ရသောစာလုံးများဖယ်ခြင်း -> Space အပိုများဖယ်ခြင်း။
၂။ **ကြိုတင်ဖြတ်တောက်ခြင်း (Pre-segmentation):** Regex (Sylbreak) ကို အသုံးပြု၍ အစောပိုင်းဖြတ်တောက်မှုပြုလုပ်ခြင်း။
၃။ **တည်းဖြတ်မှု Interface (Editor Interface):**

* မှတ်သားသူများက စစ်ဆေးခြင်းနှင့် ပြင်ဆင်ခြင်း။
* အလိုအလျောက်မှတ်ဉာဏ်စနစ်က စကားလုံးသစ်များကို မှတ်တမ်းတင်ခြင်း။
* ကိုက်ညီမှုစစ်ဆေးသည့်စနစ်က ဖြတ်တောက်မှု အမှားများကို ရှာဖွေခြင်း။
  ၄။ **ထုတ်ယူခြင်း (Export):** သန့်စင်ပြီး ကိုက်ညီမှုရှိသော ဒေတာအစုအဝေးကို ဒေါင်းလုဒ်ရယူခြင်း။

ဤကဲ့သို့ စနစ်တကျ ပြင်ဆင်ခြင်းဖြင့် Myan-Seg-Editor သည် မြန်မာဘာသာ NLP ဒေတာအစုအဝေးများ၏ အရည်အသွေးကို သိသိသာသာ မြှင့်တင်ပေးနိုင်မည်ဖြစ်သည်။

## ၉။ မြန်မာ NLP အတွက် မဟာဗျူဟာမြောက် ပေါင်းစည်းမှု (Strategic Integration for Myanmar NLP)

ဤအပိုင်းသည် `codexu/note-gen` ၏ နည်းပညာပိုင်းဆိုင်ရာ ဖွဲ့စည်းပုံနှင့် မြန်မာဘာသာစကားအတွက် အသုံးချနိုင်စွမ်းကို အခြေခံ၍ Myan-Seg-Editor အတွက် ပိုမိုနက်ရှိုင်းသော နည်းပညာဆိုင်ရာ တိုးတက်မှုများကို ဖော်ပြထားပါသည်။

### ၉.၁။ ဗိသုကာဆိုင်ရာ လုပ်ဆောင်ပုံကို မြင်သာစေခြင်း (Visualizing the Architecture)

Frontend (Next.js) မှ Backend (Rust) သို့ Data ပေးပို့ရာတွင် ကုန်ကျစရိတ် (Overhead) ကို လျှော့ချရန်အတွက် IPC Bridge ကို အောက်ပါအတိုင်း အသုံးပြုမည်:

* **Frontend (WASM):** မြန်မာစာလုံးပေါင်းသတ်ပုံ စစ်ဆေးခြင်း (Spell Check) ကဲ့သို့သော အပေါ့စားလုပ်ဆောင်ချက်များကို WebAssembly (WASM) သုံး၍ Frontend တွင်သာ လုပ်ဆောင်မည်။
* **Backend (Rust):** ဖိုင်သိမ်းဆည်းခြင်းနှင့် Vector တွက်ချက်ခြင်းကဲ့သို့သော လေးလံသော အလုပ်များကိုသာ Rust သို့ ပို့ဆောင်မည်။

### ၉.၂။ Syllable-Aware Editing အတွက် နည်းပညာဆိုင်ရာ အသေးစိတ် (Technical Refinement for Syllable-Aware Editing)

မြန်မာစာ၏ ထူးခြားချက်ဖြစ်သော ဝဏ္ဏ (Syllable) အခြေခံ တည်းဖြတ်မှုကို ပိုမိုကောင်းမွန်စေရန် **Atomic Cursor Logic** ကို အသုံးပြုမည်:

* **Atomic Cursor Logic:** Cursor သည် ယူနီကုဒ် တစ်လုံးချင်း မရွေ့ဘဲ `Intl.Segmenter` (JavaScript Native API) သို့မဟုတ် `sylbreak` logic ကို အသုံးပြု၍ Syllable Boundary များသို့သာ ခုန်ကူးမည်။
* **Visual Feedback:** `Backspace` နှိပ်လျှင် နောက်ဆုံးထည့်ထားသော Syllable တစ်ခုလုံးကို Select လုပ်ပြီး ဖျက်မလားဟု မေးမြန်းသည့် Visual Feedback စနစ် ပါဝင်မည်။

### ၉.၃။ မြန်မာစာအတွက် RAG စနစ်၏ အားနည်းချက်နှင့် ဖြေရှင်းနည်း (Refining RAG with Hybrid Search)

မြန်မာစာအတွက် Vector Search ၏ အားနည်းချက်ကို ကျော်လွှားရန် **Hybrid Search** ကို အသုံးပြုမည်:

* **Vector Search:** အဓိပ္ပာယ် တူညီမှုကို ရှာရန် (Semantic Meaning)။
* **Keyword Search (BM25):** အတိအကျ စာလုံးပေါင်းနှင့် မြန်မာစာတွင် သီးခြားအမည်များကို ရှာဖွေရာတွင် ပိုမိုတိကျစေရန် Keyword Search ကို တွဲဖက်အသုံးပြုမည်။

### ၉.၄။ MCP နှင့် မြန်မာ NLP Server ပေါင်းစပ်ပုံ (Deep Dive into MCP Integration)

"Myanmar NLP MCP Server" ကို အောက်ပါအတိုင်း စနစ်ကျစွာ တည်ဆောက်မည်:

* **MCP Host (Myan-Seg-Editor App):** အသုံးပြုသူ၏ တောင်းဆိုချက်ကို လက်ခံသည်။
* **StdIO Transport:** Latency လျှော့ချရန် Process များအကြား တိုက်ရိုက်ချိတ်ဆက်သည်။
* **NLP Server (Python/Rust):**
  * `tools/segment`: စာကြောင်းလိုက် ဖြတ်ပေးခြင်း။
  * `tools/tokenize`: Syllable အလိုက် ဖြတ်ပေးခြင်း။
  * `tools/g2p`: Text-to-Speech အတွက် အသံထွက် (Phoneme) ပြောင်းပေးခြင်း။

ဤဗိသုကာပုံစံသည် အနာဂတ်တွင် မြန်မာစာ စစ်ဆေးခြင်း (Grammar Checker) ကဲ့သို့သော AI လုပ်ဆောင်ချက်များကို Main App ကို ပြင်ဆင်စရာမလိုဘဲ နောက်ဆက်တွဲ Module အနေဖြင့် လွယ်ကူစွာ ထပ်ဖြည့်နိုင်စေမည် ဖြစ်သည်။

---

ဤဖြည့်စွက်ချက်များသည် Myan-Seg-Editor ကို သာမန် မှတ်စု App အဆင့်မှ မြန်မာစာကို နားလည်သော **"ဉာဏ်ရည်မြင့် ဒစ်ဂျစ်တယ် လက်ထောက်"** အဖြစ်သို့ ပြောင်းလဲပေးနိုင်မည့် လက်တွေ့ကျသော နည်းပညာလမ်းကြောင်းများ ဖြစ်ပါသည်။

## ၁၀။ အသုံးပြုသူ အခန်းကဏ္ဍများနှင့် ခွင့်ပြုချက်များ (User Roles & Permissions)

စကေးကြီးမားသော မှတ်သားမှုစီမံကိန်းများကို စီမံခန့်ခွဲရန်၊ ဤစနစ်သည် အသုံးပြုသူ အခန်းကဏ္ဍ ၃ မျိုးကို ပံ့ပိုးပေးသည်-

* **အက်ဒမင် (Admin):** စီမံကိန်းစီမံခန့်ခွဲမှု၊ အသုံးပြုသူစီမံခန့်ခွဲမှုနှင့် ဒေတာထုတ်ယူမှုအတွက် အပြည့်အဝဝင်ရောက်ခွင့်။ စီမံကိန်းတစ်ခုလုံးအတွက် အနုစိတ်နှုန်း စည်းမျဉ်းများကို ပြင်ဆင်နိုင်သည်။
* **ပြန်လည်သုံးသပ်သူ (Reviewer - Senior Annotator):** မှတ်သားသူများ ပြုလုပ်ထားသော မှတ်သားမှုများကို ပြန်လည်သုံးသပ်၍ အတည်ပြုခြင်း/ပယ်ချခြင်း ပြုလုပ်နိုင်သည်။ "သံသယဖြစ်ဖွယ် ပဋိပက္ခများ" ကို ဖြေရှင်းရန် အာဏာအပ်နှင်းခံရသည်။
* **မှတ်သားသူ (Annotator):** စကားလုံးဖြတ်တောက်ခြင်းနှင့် စာလုံးပေါင်းသတ်ပုံပြင်ဆင်ခြင်း ပြုလုပ်ရန် အဓိကအခန်းကဏ္ဍ။

## ၁၁။ ပြင်ဆင်ခြင်းနှင့် ထိန်းသိမ်းခြင်း (Deployment & Maintenance)

* **ပြင်ဆင်ခြင်း (Deployment):** Docker အခြေခံ containerization ကို အသုံးပြု၍ ဒေသတွင်းဆာဗာများ သို့မဟုတ် cloud platforms (AWS/GCP) တွင် လွယ်ကူစွာ ပြင်ဆင်နိုင်သည်။
* **စောင့်ကြည့်ခြင်း (Monitoring):** မှတ်သားသူတစ်ဦးချင်းစီ၏ မှတ်သားမှုအမြန်နှုန်းနှင့် အမှားနှုန်းများကို ခြေရာခံရန် ပေါင်းစပ်ထားသော မှတ်တမ်းများ။
* **အရန်သိမ်းခြင်း (Backup):** ဒေတာဆုံးရှုံးမှုကို ကာကွယ်ရန် JSON ဒေတာနှင့် ဝေါဟာရစာရင်း၏ နေ့စဉ် အလိုအလျောက်အရန်သိမ်းခြင်း။

## ၁၂။ အနာဂတ်တိုးတက်မှုများ (Future Enhancements)

* **LLM-အကူအညီဖြင့် ပြန်လည်ပြင်ဆင်ခြင်း (LLM-Assisted Refinement):** Large Language Models (GPT-4 ကဲ့သို့သော သို့မဟုတ် မြန်မာဘာသာစကားအတွက် အထူးပြုထားသော LLM များ) ကို ပိုမိုတိကျသော ကြိုတင်ဖြတ်တောက်ခြင်းနှင့် အခြေအနေအလိုက်အကြံပြုချက်များ ပေးရန် ပေါင်းစပ်ထည့်သွင်းမည်။
* **Active Learning:** စနစ်က ကိုယ်တိုင်ယုံကြည်မှုအနည်းဆုံးဖြစ်သည့် စာကြောင်းများကို လက်ဖြင့်ပြန်လည်သုံးသပ်ရန် ဦးစားပေးသည့် active learning loops ကို အကောင်အထည်ဖော်မည်။
* **ဘာသာစကားမျိုးစုံပံ့ပိုးမှု (Multi-language Support):** အလားတူစာလုံးပုံစံလက္ခဏာများ (ဥပမာ - ခမာ၊ လာအို) ရှိသည့် အခြားသော အရင်းအမြစ်နည်းပါးသည့် ဘာသာစကားများကို ပံ့ပိုးရန် ဤဘောင်ကို ချဲ့ထွင်မည်။

## ၁၃။ API ဒီဇိုင်း (Internal) (API Design - Internal)

React frontend နှင့် Python backend အကြား ဆက်သွယ်ဆောင်ရွက်မှုကို လွယ်ကူစေရန်၊ အောက်ပါအဓိက API endpoints များကို သတ်မှတ်ထားသည်-

* **POST `/api/segment`**: မူရင်းစာသားကို လက်ခံပြီး Sylbreak+ algorithm ကို အသုံးပြု၍ ဝဏ္ဏ/စကားလုံး block များစာရင်းကို ပြန်ပေးသည်။
* **POST `/api/check-conflicts`**: ဖြတ်တောက်မှု ပြောင်းလဲတိုင်းတွင် trigger လုပ်သည်။ လက်ရှိစီမံကိန်းအတွင်း မကိုက်ညီမှုများကို scan ဖတ်သည်။
* **GET/POST `/api/glossary`**: အဆင့်လိုက်မှတ်ဉာဏ်စနစ် (Manual/Auto) ၏ ရယူခြင်းနှင့် အပ်ဒိတ်လုပ်ခြင်းကို စီမံခန့်ခွဲသည်။
* **GET `/api/stats`**: Dashboard အတွက် အချိန်နှင့်တပြေးညီ တိုးတက်မှုအတိုင်းအတာများကို ပြန်ပေးသည်။

## ၁၄။ စွမ်းဆောင်ရည်နှင့် စကေးချဲ့နိုင်မှု (Performance & Scalability)

* **Virtualized Rendering:** ဒေတာအစုအဝေးသည် စာကြောင်း ၅,၀၀၀ ကျော်လွန်ပါက၊ တည်းဖြတ်ဆော့ဖ်ဝဲသည် windowing/virtualization (ဥပမာ `react-window`) ကို အသုံးပြု၍ UI သည် တုံ့ပြန်မှုရှိစေရန် မြင်နိုင်သော block များကိုသာ render လုပ်သည်။
* **Asynchronous Scanning:** ပဋိပက္ခတွေ့ရှိမှုသည် UI ကို block မဖြစ်စေရန် background worker (Celery သို့မဟုတ် အလားတူ) အဖြစ် လည်ပတ်သည်။
* **Regex Optimization:** မြန်မာဖြတ်တောက်ခြင်း Regex ကို ကြိုတင်ပြင်ဆင်ပြီး cache လုပ်ထားကာ အချိန်နှင့်တပြေးညီ တည်းဖြတ်မှုအတွင်း overhead ကို လျှော့ချသည်။

## ၁၅။ လုံခြုံရေးနှင့် ဒေတာလျှို့ဝှက်မှု (Security & Data Privacy)

* **Authentication:** JWT အခြေခံ authentication ကို လုံခြုံသော login နှင့် session စီမံခန့်ခွဲမှုအတွက် အသုံးပြုသည်။
* **Data Isolation:** အမျိုးမျိုးသော စီမံကိန်းများ သို့မဟုတ် အဖွဲ့အစည်းများသည် တစ်ခုနှင့်တစ်ခု၏ ဒေတာများကို ဝင်ရောက်ကြည့်ရှု၍မရစေရန် multi-tenancy ပံ့ပိုးမှု။
* **Audit Logs:** ဒေတာတည်ငြိမ်မှုကို ထိန်းသိမ်းရန် ဒေတာထုတ်ယူခြင်းနှင့် ဖျက်သိမ်းခြင်းလုပ်ဆောင်ချက်အားလုံး၏ အသေးစိတ်မှတ်တမ်းများ။

## ၂၄။ ဝဏ္ဏ ဖြတ်တောက်ခြင်း Algorithm: နည်းပညာပိုင်း အသေးစိတ်ရှင်းလင်းချက် (Syllable Segmentation Algorithm: Technical Deep Dive)

မြန်မာ ဝဏ္ဏဖြတ်တောက်ခြင်း algorithm သည် Myan-Seg-Editor ၏ "Smart Splitting" လုပ်ဆောင်ချက်၏ အခြေခံအုတ်မြစ် (Foundational Component) ဖြစ်သည်။ ဤအပိုင်းတွင် အကောင်အထည်ဖော်ထားသော နည်းပညာပိုင်း အသေးစိတ် ရှင်းလင်းချက်ကို ပေးအပ်ထားပါသည်။

### ၂၄.၁။ Algorithm ခြုံငုံသုံးသပ်ချက်

အကောင်အထည်ဖော်ထားသော algorithm သည် [`myWord` repository](https://github.com/sithu015/myWord) မှ **substitution-based regex approach** အပေါ် အခြေခံထားပြီး၊ အထူးသဖြင့် `syl_segment.py` မှ logic ကို ပြောင်းရေးထားခြင်း ဖြစ်သည်။ ဤချဉ်းကပ်နည်းသည် pattern-matching approaches များနှင့် အခြေခံအားဖြင့် ကွဲပြားပါသည်-

**❌ မမှန်ကန်သော ချဉ်းကပ်နည်း (Pattern Matching):**
```python
# ဝဏ္ဏပြည့်စုံမှုကို ရှာဖွေကြိုးစားခြင်း
syllable_pattern = r'[consonant][medial]?[vowel]?[final]?'
syllables = re.findall(syllable_pattern, text)  # မပြည့်စုံသော ရလဒ်များ ပြန်ပေးသည်
```

**✅ မှန်ကန်သော ချဉ်းကပ်နည်း (Substitution-Based):**
```typescript
// ဝဏ္ဏနယ်နိမိတ်များ မတိုင်မီ delimiters များ ထည့်သွင်းခြင်း
const broken = text.replace(BreakPattern, DELIMITER + '$1');
const syllables = broken.split(DELIMITER).filter(s => s.length > 0);
```

### ၂၄.၂။ အဓိကအကောင်အထည်ဖော်မှု (Core Implementation)

Algorithm သည် အဓိကအဆင့် ၃ ဆင့် ပါဝင်သည်-

#### အဆင့် ၁: စာသားသန့်စင်ခြင်း (Text Cleaning)
```typescript
function cleanText(text: string): string {
    // Zero-Width Space (U+200B) ဖယ်ရှားခြင်း
    let cleaned = text.replace(/\u200B/g, '');
    // အခြား မမြင်ရသော control characters များ ဖယ်ရှားခြင်း
    cleaned = cleaned.replace(/[\u200C\u200D\uFEFF]/g, '');
    return cleaned;
}
```

```typescript
// Break pattern: အောက်ပါအရာများ မတိုင်မီ delimiter ထည့်သွင်းပါ-
// ၁။ ဗျည်း (stacked symbol အပြီးတွင် မဟုတ်ခြင်း၊ aThat/ssSymbol အရှေ့တွင် မဟုတ်ခြင်း)
// ၂။ အင်္ဂလိပ်/ပန်ချူရေးရှင်း characters များ
const BreakPattern = new RegExp(
    '(' +
        '(?<!' + ssSymbol + ')[' + myConsonant + '](?![' + aThat + ssSymbol + '])' +
        '|' +
        '[' + enChar + otherChar + ']' +
    ')',
    'gu'  // global + unicode
);
```

#### အဆင့် ၃: ဖြတ်တောက်ခြင်း (Segmentation)
```typescript
export function segmentIntoSyllables(text: string): string[] {
    const cleaned = cleanText(text);
    const broken = cleaned.replace(BreakPattern, DELIMITER + '$1');
    return broken.split(DELIMITER).filter(s => s.length > 0);
}
```

### ၂၄.၃။ Pattern အသေးစိတ်ခွဲခြမ်းစိတ်ဖြာချက် (Pattern Breakdown)

`BreakPattern` regex သည် **negative lookbehind** နှင့် **negative lookahead** ကို အသုံးပြု၍ ဝဏ္ဏနယ်နိမိတ်များကို ရှာဖွေသည်-

```typescript
(?<!ssSymbol)[myConsonant](?![aThat|ssSymbol])
```

**အဓိပ္ပာယ်ဖော်ပြချက်:**
- `(?<!...)` = Negative lookbehind: "အရှေ့တွင် ... မရှိခြင်း"
- `(?!...)` = Negative lookahead: "အနောက်တွင် ... မရှိခြင်း"
- **ဗျည်းသည် ဝဏ္ဏကို ခွဲခြားသည် သို့သော်:**
  - Stacked consonant symbol `္` (U+1039) အပြီးတွင် ရှိခြင်း မဟုတ်ရပါ
  - aThat `်` (U+103A) သို့မဟုတ် အခြား stacked symbol အရှေ့တွင် ရှိခြင်း မဟုတ်ရပါ

**ဥပမာများ:**

| Input | ခွဲထားသောပုံစံ | ဝဏ္ဏများ | ရှင်းလင်းချက် |
|-------|-------------|-----------|------------|
| `ကျောင်း` | `\|က\|ျောင်း` | `['က','ျောင်း']` | `က` နှင့် `ျ` အရှေ့တွင် ခွဲသည် |
| `သား` | `\|သား` | `['သား']` | ဝဏ္ဏတစ်ခုတည်း |
| `ဆရာဝန်` | `\|ဆ\|ရာ\|ဝန်` | `['ဆ','ရာ','ဝန်']` | ဝဏ္ဏသုံးခု |
| `ကျွန်တော်` | `\|ကျွန်\|တော်` | `['ကျွန်','တော်']` | Medials များသည် ဗျည်းနှင့် တွဲထားသည် |

### ၂၄.၄။ Stacked Consonants ကိုင်တွယ်ခြင်း

Stacked consonants (ဥပမာ `င်္ဂ` = င + ္ + ဂ) များသည် တစ်ခုတည်းသော ယူနစ်အဖြစ် အတူတကွ ရှိနေရမည်-

```typescript

// ဤသည်က stacked consonant marker အပြီးတွင် ခွဲခြားခြင်းကို ကာကွယ်သည်

// ဥပမာ: "အင်္ဂလိပ်"
// အဆင့် ၁: သန့်စင်ခြင်း
// အဆင့် ၂: Break pattern အသုံးပြုခြင်း
//   - 'အ' အရှေ့တွင်: ခွဲသည်
//   - 'င' အရှေ့တွင်: ခွဲသည်
//   - 'ဂ' အရှေ့တွင်: မခွဲပါ (္ အပြီးတွင် ရှိသည်)

//   - 'ပ' အရှေ့တွင်: ခွဲသည်

```

### ၂၄.၅။ ပန်ချူရေးရှင်းနှင့် ရောနှောပါဝင်သောအကြောင်းအရာ (Punctuation and Mixed Content)

အင်္ဂလိပ် characters များနှင့် punctuation များကို သီးခြား segments များအဖြစ် ကိုင်တွယ်သည်-

### ၂၄.၆။ Editor နှင့် ပေါင်းစပ်ခြင်း (Integration with Editor)

ဝဏ္ဏဖြတ်တောက်ခြင်းသည် Segmentation Mode တွင် **Smart Space** လုပ်ဆောင်ချက်ကို စွမ်းအားပေးသည်-

```typescript
// EditorWorkspace.tsx တွင်
case ' ':  // Space key နှိပ်ခြင်း
    e.preventDefault();
    if (currentSeg.text.length > 1) {
        const syllables = segmentIntoSyllables(currentSeg.text);
        if (syllables.length > 1) {
            // ပထမဝဏ္ဏနယ်နိမိတ်တွင် ခွဲသည်
            const splitPos = syllables[0].length;
            splitSegment(currentLineIndex, currentSegmentIndex, splitPos);
        }
    }
    break;
```

**အသုံးပြုသူအတွေ့အကြုံ:**
1. အသုံးပြုသူသည် segment `"ကျောင်းသား"` (ကျောင်းသား) သို့ ရွှေ့သည်
2. **Space** နှိပ်သည် → စနစ်က `segmentIntoSyllables("ကျောင်းသား")` ကို ခေါ်သည်
3. Algorithm က `['ကျောင်း', 'သား']` ပြန်ပေးသည်
4. Editor က position 9 (ပထမဝဏ္ဏ၏ အရှည်) တွင် ခွဲသည်
5. ရလဒ်: Segments နှစ်ခု `"ကျောင်း"` နှင့် `"သား"`

### ၂၄.၁၀။ အနာဂတ်တိုးတက်မှုများ (Future Enhancements)

Algorithm အတွက် ဖြစ်နိုင်သော တိုးတက်မှုများ-

1. **Cluster Analysis**: `"ကျောင်းသား"` (student) ကဲ့သို့သော compound words များကို ရှာဖွေခြင်း (အတူတကွ ရှိနေသင့်သည်)

3. **Adaptive Delimiters**: သဒ္ဒါစည်းမျဉ်းတစ်ခုချင်းစီအလိုက် အသုံးပြုသူ သတ်မှတ်ထားသော break preferences များကို ပံ့ပိုးခြင်း
4. **Performance Metrics**: စာရွက်ကြီးများ (ဝဏ္ဏ ၁၀,၀၀၀+ ) အတွက် profiling ထည့်သွင်းခြင်း
